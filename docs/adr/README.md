# Architecture Decision Records (ADRs)

This folder contains ADRs documenting architectural choices. Each ADR follows the short format: title, date, status, context, decision, consequences, and alternatives.

Add new ADRs by creating a numbered markdown file (e.g. `0005-my-decision.md`) and commit it to git. When replacing an ADR, mark the old one `Superseded` and reference the new ADR.
